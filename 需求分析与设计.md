#### 需求分析与设计

对于工程师来说，如果要最求长远的发展，不能一直只把自己放在执行者的角色，不能只是一个代码实现者，还要有独立负责一个系统的能力，能端到端开发一个完整的系统。

包括：前期的需求沟通分析、中期的代码设计实现、后期的系统上线维护等。

对于产品设计、需求分析，我们要学会“借鉴”，一定不要自己闷头想。
一方面这样做很难想全面，另一方面从零开始设计也比较浪费时间。


#### 系统设计

1. 合理地将功能划分到不同模块

把合适的代码放到合适的类中。
将合适的功能放到合适的模块中。
合理划分模块可以做到模块层面的高内聚、低耦合，架构整洁清晰。

2. 设计模块与模块之间的交互关系


3. 设计模块的接口、数据库、业务模型




#### 业务开发

1. 接口设计

2. 数据库设计

3. 业务模型设计

数据库和接口的设计非常重要，一旦设计好并投入使用之后，这两部分都不能轻易改动。

改动数据库表结构，需要涉及数据的迁移和适配；

改动接口，需要推动接口的使用者作相应的代码修改。


#### 为什么要分 MVC 三层开发？

* Controller 

Controller 层负责接口暴露

* Service 

Service 层负责核心业务逻辑（业务模型）

* Repository

Repository 层负责数据读写


1. 分层能起到代码复用的作用

2. 分层能起到隔离变化的作用

3. 分层能起到隔离关注点的作用

4. 分层能提高代码的可测试性

5. 分层能应对系统的复杂性


#### BO、VO、Entity 存在的意义？

View Object 对应 Controller 层

Business Object 对应 Service 层

Entity 对应 Repository 层
